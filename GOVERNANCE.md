# Governance

Twill DB is currently **maintainer-led**.

The core architecture, roadmap, and all merge decisions are managed exclusively
by the project maintainer, Reza Baita.

- **Source is public.** The full implementation and design specs are published
  for anyone to read, evaluate, and self-host.
- **Feedback is welcome.** Issues, bug reports, architecture discussion, and
  feature suggestions are encouraged.
- **Contributions are currently closed.** Commit access and external code
  contributions are restricted; pull requests are not being accepted and may be
  closed without review (see [`.github/CONTRIBUTING.md`](.github/CONTRIBUTING.md)).
- **Licensing.** Twill DB ships under the Business Source License 1.1, a
  source-available license that permits self-hosting (including in production)
  but reserves the right to offer Twill DB as a managed/hosted database service.
  See [`LICENSE`](LICENSE).

This model — build in public, develop in private, and defer opening
contributions until the architecture stabilizes — is deliberate for a database
project at this stage. We expect to revisit both the contribution policy and the
governance model as the project matures toward a stable `1.0`.
